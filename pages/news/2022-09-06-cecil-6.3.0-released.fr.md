### Fonctionnalités

#### Meilleure gestion des variables de premier plan

Les variables de présentation _simples_ (c'est-à-dire : titre, langue, etc.) sont stockées dans une propriété et les valeurs _complexes_ (comme les collections) sont stockées dans un attribut d'objet dédié.

Cela n'a aucun impact sur les appels au sein d'un modèle.

_Exemple:_

```twig
{{ page.language }} # <=> $page->getVariable('language');
{{ page.pages }} # <=> $page->getPages();
```

### Correctifs

- Balise de fin manquante pour l'élément principal du modèle par défaut corrigée (par @[magentix](https://github.com/magentix))
- [#1456](https://github.com/Cecilapp/Cecil/issues/1456) corrigé (commande `new:page`)
- Le poids de l'entrée dans le menu est forcé d'être une valeur entière
- Le nom de fichier `new:page` peut désormais contenir plusieurs points dans son nom
- L'URL d'une page d'alias est désormais localisée
- La commande `self-update` fonctionne à nouveau grâce au fork de [_laravel-zero/phar-updater_](https://github.com/laravel-zero/phar-updater)
